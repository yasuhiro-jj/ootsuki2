# Customer Memory and QR Login MVP

**Status:** Implemented MVP  
**Date:** 2026-07-13

## Purpose

The chatbot can now keep a pseudonymous customer identity across visits without
requiring names, phone numbers, LINE accounts, or other direct personal
information.

The goal is to support warmer repeat-visit conversations later, such as
remembering preferences and previous orders, while keeping the first connection
safe and narrow.

## Current MVP

Flow:

```text
QR or browser access
-> frontend checks localStorage
-> backend issues or restores anonymous_customer_id
-> frontend stores anonymous_customer_id locally
-> /session and /chat send customer_id
-> backend links session to that pseudonymous customer_id
```

Backend endpoint:

```text
POST /customer-memory/identify
```

Request:

```json
{
  "anonymous_customer_id": "anon_existing_id_or_null",
  "consent_accepted": false,
  "source": "chat_window"
}
```

Storage:

```text
outputs/customer_memory_profiles.json
```

Stored profile fields:

```text
customer_profile_id
anonymous_customer_id
consent_status
preference_tags
favorite_items
avoided_items
last_ordered_items
declined_products
visit_count
last_visit_at
communication_notes
```

## Privacy Boundary

The MVP does not store:

- real name
- phone number
- email address
- LINE account id
- address
- family member identity
- sensitive personal attributes
- permanent full conversation transcript

The anonymous id uses the `anon_` prefix and is generated by the backend.
Invalid or user-supplied non-anonymous ids are rejected and replaced.

## Consent

The backend stores `consent_status`.

Current values:

```text
unknown
consented
```

The frontend keeps the consent flag separately in localStorage. The first MVP
does not turn customer memory into automatic sales suggestions.

## Current Chatbot Connection

The chatbot already had `customer_id` on `ChatRequest` and session creation.
This MVP reuses that field and passes the pseudonymous id from the frontend.

The customer memory profile is not yet used to alter production replies.

## Out of Scope

- automatic upsell based on customer memory
- QR account registration with real identity
- LINE account integration
- POS integration
- long-term full conversation storage
- AI-generated personality labels
- family or companion identification

## Next Phase

Recommended next steps:

1. Add a small consent UI.
2. Save explicit favorite and avoided items only after consent.
3. Add customer-memory summaries from confirmed order events.
4. Let recommendation rules read `avoided_items` before suggesting products.
5. Add admin review tools before using memory in customer-facing replies.
